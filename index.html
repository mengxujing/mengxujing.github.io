用hexo github搭建博客踩过的坑

说一说我搭建这个hexo+github博客时出现的问题以及我的解决方法

创建完成后直接输入http://username.github.io 却出现404错误

  • 创建一个新的仓库

  • 给仓库命名

  • 进入到这个仓库，复制这个链接


  • 打开blog文件夹，找—config.yml


  • 更改-config.yml的配置

  • 在blog目录下执行如下命令

    ‘’’ bash
    //生成静文件
    hexo g
    //部署网站
    hexo d
    ‘’’

  • 访问http//username.github.io网址就可以看到博客界面了。

    hexo更换主题后，本地运行测试命令行提示 : WARN No layout: index.html?

  • 检查下载的主题是不是放在了themes文件夹下面，比如next主题的放置，就是点开themes就看到next的文件夹，next下面就是主题的文件，不能打开next文件夹还有themes

  • 检查站点配置_config.yml是不是下面的形式

theme: 你的主题名字

注意：

  • 这里是theme，不是themes.
  • theme后面与主题名直接有一个空格。
  • 主题名字是否与themes文件夹下 主题的名字一致。

hexo指令无法使用，一直报错，不管是hexo clean 、hexo g、还是hexo d 、hexo new，这些指令都出现了错误

  • 报错原因：插件错误，hexo-deployer-git插件出现错误
  • 在blog下node_modules文件夹下检查“hexo-deployer-git”文件是否异常

    npm WARN optional SKIPPING OPTIONAL DEPENDENCY: fsevents@1.2.0 (node_modules\fsevents

  • 原因：
    fsevents不在package.json里，但是仍然安装了，是因为你的系统不是Windows系统，fsevents是苹果系统的可选依赖,你的项目有可能是团队项目，别人在他的mac上安装了fsevents相关依赖库，所以到这边你也就安装到你的windows上边了。你可以检查你的package.json 文件中是不是有fsevents相关依赖，删除即好！

    如果没有，其他的npm包也会有依赖fsevents的！！！

    这是warning错误，是因为mac下需要 fsevents，这里是在windows环境，所以可以忽略这个错误。

面向对象的三大特征

1. 封装
2. 继承
3. 多态

封装

  1. 概念：

      指的就是将描述实体的数据和对这些数据的控制进行操作集合到一起，形成一个封装体。对类内的某些信息进行隐藏，不允许外界进行直接访问，但外部可以通过该类提供的方法实现对该类的操作和访问。保证了程序的安全性的可维护性。
  2. 优点：

    有效的减少耦合性;

    在类的内部可以自由修改;

    可以对成员变量进行精确的控制;

    隐藏信息，实现细节。
  3. 实现方法：

       封装就是通过修改属性的可见性（修改为privated),设置getter/setter方法（用于属性的读写）以及在这些方法中加入属性控制语句（对属性值的合法性进行判断)。

    ‘’’
    public class Person{
    private String name;
    private int age;
    ​
    public int getAge(){
    return age;
    }
    ​
    public String getName(){
    return name;
    }
    ​
    public void setAge(int age){
    this.age = age;
    }
    ​
    public void setName(String name){
    this.name = name;
    }
    }
    ‘’’

      this 关键字可以调用自身类和方法，也可以解决实例变量（private String name）和局部变量（setName(String name)中的name变量）之间发生的同名的冲突问题。

继承

  1. 概念：

      继承就是子类继承父类的特征和行为，使得子类对象（实例）具有父类的实例域和方法，或子类从父类继承方法，使得子类具有父类相同的行为。

    （其实就是从已有的类派生出新的类，为继承。
    当在不同的类中也出现共同的特征和动作，可以把这些共同的特征和动作放在一个类中，成为父类。然后将其扩展为其它多个子类，这些子类继承父类中的特征和动作。继承的好处就是是代码变得简洁，易于维护，易于理解。）

  2. 特点：

    子类继承父类非 private 的属性、方法。

    子类可以对父类进行扩展。

    Java 的继承是单继承，但是可以多重继承。

    提高了类之间的耦合性（代码独立性越差）。

3.
‘’’
class Animal {
void move() {
System.out.println(“animal : move”);
}
}

class Dog extends Animal {
void move() {
   System.out.println("dog : move");
 }
void eatTest() {
  this.move();   // this 调用自己的方法
  super.move();  // super 调用父类方法
}
}

public class Test {
public static void main(String[] args) {
Dog d = new Dog();
d.eatTest();
}
}
'''

super：用来引用当前对象的父类。

  • super 还可以调用父类的构造函数。但super语句必须是子类构造方法的第一条语句。不能在子类中使用父类构造方法名来调用父类构造方法。 且父类的构造方法不被子类继承。调用父类的构造方法,如果父类的构造方法中带有参数，则必须在子类的构造方法中通过 super 关键字调用父类的构造方法并配以适当的参数列表。如果父类构造方法没有参数，系统会自动调用父类的无参构造方法。
  • 静态方法中不能使用 super 关键字。

    this：指向自己的引用。
  1. final 关键字
    声明类可以把类定义为不能继承的，即最终类；或者用于修饰方法，则该方法不能被子类重写.
  • final修饰类中的方法,方法可以被继承，但继承后不能被重写。
  • final修饰类，类不可以被继承。

多态

多态就是同一个行为具有多个不同表现形式或形态的能力。同一个类使用不同的实例对象执行的不同的操作。

多态存在的三个必要条件:

  • 继承

  • 重写

  • 父类引用指向子类对象(Father a=new Son)

    (Son型自动提升为Fatherl型（向上转型，得按Father对象进行操作），因此a只能调用Father类中的方法，对Son中特有功能无法访问 )。如果还想用Son的特有功能，可以强制向下转型：Son c = (Son) a。向下转型是为了使用特有方法。如果两者不具备继承关系不能强制转换！（向上转型为了提高扩展性并限制特有方法，向下转型为了使用特定方法）。对于转型自始至终都是子类对象做着类型的变化。


    [引用变量在编译阶段只能调用编译时类型所具有的方法，但运行时则执行他运行时类型所具有的方法。]
